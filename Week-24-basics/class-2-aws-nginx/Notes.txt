Reverse Proxies (Why need Nginx?)

VPN is a Forward Proxy. 
http://my-website.com is automatically redirects to http://my-website.com:80
IP:Port is not good looking (e.g., https://google.com:443 vs https://google.com).

Try Running Node on Port 80 -> Permission Denied! 
You can't run multiple processes on port 80. As Express server is already running on port 80.

if you sell clothes and shoes. 
sellcloths.com running on port 3000, sellshoes.com running on port 3001. 
Both need port 80 for HTTP. 
Nginx works as a reverse proxy.

Architecture:
EC2 Server -> Nginx (Port 80) -> routes cloths.com to node index.js (Port 3000) and shoe.com to node index.js (Port 3001).
